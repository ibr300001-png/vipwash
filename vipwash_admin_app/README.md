
# VIP Wash Admin (Mobile-First)

مشروع مستقل تمامًا عن موقع العملاء — لكنه يقرأ نفس قواعد البيانات مباشرة (بدون أي تعديل على موقع العملاء).

## التشغيل المحلي
```bash
pip install -r requirements.txt
# لو كانت قواعد البيانات بجانب app.py بنفس المجلد:
export LOYALTY_API_DB=$(pwd)/loyalty_api.db
export LOYALTY_DB=$(pwd)/loyalty.db
python app.py
# ثم افتح: http://127.0.0.1:5000/admin
```

> أول دخول:
> - ADMIN: qaaz / 1234
> - ACCOUNTANT: amee / 1234

## النشر (Render أو VPS)
- ارفع هذا المشروع كما هو.
- ضع ملفات قواعد البيانات `loyalty_api.db` و `loyalty.db` بجانب `app.py`
  أو عيّن متغيرات البيئة:
  - `LOYALTY_API_DB=/path/to/loyalty_api.db`
  - `LOYALTY_DB=/path/to/loyalty.db`

## الميزات
- صفحة تسجيل دخول (موبايل).
- لوحة الإدارة:
  - قارئ QR + إدخال يدوي (TOKEN).
  - إحصائيات (المحاسبين، التقييمات، أوقات الذروة).
  - تنبيهات حسب قواعدك (قابلة للنقر لعرض بيانات العميل).
  - زر إعدادات (⚙️) كشيت سفلي: نقاط لكل زيارة + حد الاستبدال + إدارة المستخدمين (إضافة/تعديل/حذف).
  - العملاء: جدول كامل + بحث + تصدير PDF.
  - السحب العشوائي: زر واحد → اختيار فائز + **QR** + زر واتساب أعمال (النص جاهز).
- شاشة المحاسب:
  - قارئ QR + إدخال يدوي + نافذة بيانات العميل + اعتماد/استبدال (الاستبدال يتفعّل تلقائياً عند بلوغ الحد).
- لغتين + ثيم ليلي/نهاري.
- الهوية: تم استخدام `assets/logo.png` (ضع لوقو المغسلة بهذا الاسم).

> ملاحظة: واتساب ويب لا يسمح بإرفاق صورة تلقائيًا عبر الرابط؛ افتح الرسالة من الزر ثم أرفق صورة الـQR التي نزّلتها.
